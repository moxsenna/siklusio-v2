# Siklusio Database Guide

Tanggal audit terakhir: 2026-06-03.

Dokumen ini adalah panduan database untuk developer. Tujuannya adalah memastikan
seluruh perubahan skema dikelola melalui migrasi terstruktur, bukan perubahan manual.

---

## Source Of Truth

`supabase/migrations/` adalah source of truth untuk seluruh perubahan skema database production.

Aturan kerja:

1. Buat berkas migrasi baru menggunakan CLI: `supabase migration new <nama_deskriptif>`.
2. Edit file migrasi yang dihasilkan, jangan membuat timestamp manual.
3. Jalankan `npm run db:push:dry-run` sebelum melakukan apply ke remote database.
4. Setelah migrasi diaplikasikan ke database remote, jalankan `npm run db:types` untuk memperbarui type definitions.
5. Commit berkas migrasi, generated types, dan update dokumentasi ini dalam satu perubahan.

## Legacy Root SQL

Berkas `.sql` di folder root `supabase/` adalah legacy/manual reference.
Mereka berguna sebagai acuan sejarah fitur, namun bukan alur rilis produksi.

Contoh file legacy/reference:

| File | Status |
| --- | --- |
| `supabase/schema.sql` | Bootstrap/core reference untuk `profiles` dan `activity_history` |
| `supabase/community*.sql` | Reference untuk community tables, privacy, RPC admin, avatar, dan comments |
| `supabase/affiliates.sql`, `affiliate_conversions.sql` | Reference affiliate dan payout tracking |
| `supabase/pending_registrations.sql`, `checkout_sessions.sql` | Reference checkout/payment support |
| `supabase/activity_history_sync_hardening.sql` | Reference trigger `updated_at` activity history |
| `supabase/community_verify.sql` | Diagnostic SQL, bukan DDL migration |

Rule: jangan copy-paste root SQL langsung ke database production. Selalu buat migrasi
CLI (`supabase migration new`) dan uji dry-run.

## Security Guidelines

- Semua tabel publik wajib mengaktifkan Row Level Security (RLS).
- Fungsi dengan tag `SECURITY DEFINER` wajib ditinjau karena bypass RLS.
- View yang dapat diakses klien harus menggunakan `security_invoker = true` pada PostgreSQL 15+
  atau dilindungi policy/grant yang sesuai.
- Hak akses fungsi diperketat (Phase 28): AI credit mutations & affiliate helper dibatasi untuk
  `service_role` saja, community/admin RPC hanya untuk user terautentikasi (`authenticated`),
  akses `anon` dicabut, dan fungsi helper `is_admin(uid)` diamankan.

## Commands

Berikut perintah utama untuk verifikasi dan migrasi database:

```bash
# Cek list status migrasi database
npm run db:migrations:list

# Lakukan uji coba push migrasi (dry-run)
npm run db:push:dry-run

# Jalankan linter skema database
npm run db:lint

# Update type definitions TypeScript
npm run db:types
```

Aturan rilis:
- `db:push:dry-run` aman dijalankan kapan saja karena tidak mengubah database.
- Jangan menjalankan push aktual (`supabase db push`) tanpa persetujuan deployment.

## Long-Term Cleanup

Rekomendasi cleanup dan perbaikan di masa mendatang:

1. Buat baseline/squash untuk berkas root SQL legacy.
2. Tambahkan integration test untuk kebijakan RLS menggunakan user test non-admin dan admin.
3. Tinjau seluruh database view sebelum menambahkan API surface baru (gunakan security-invoker views).
4. Gunakan linting database (`npm run db:lint`) sebagai release gate wajib di CI/CD.
